## 对比分析技能

### 核心框架
1. **维度定义**: 确定比较的关键维度
2. **信息收集**: 为每个选项收集各维度数据
3. **标准化评估**: 使用统一标准评估
4. **权重分配**: 根据用户需求调整权重
5. **综合判断**: 给出推荐和理由

### 常用比较维度
- 功能/特性
- 性能/效率
- 成本/价格
- 易用性/学习曲线
- 生态系统/社区支持
- 可扩展性/灵活性
- 安全性/稳定性

### 输出格式
```
【比较对象】A vs B [vs C...]

【比较维度】
| 维度 | A | B | 说明 |
|------|---|---|------|
| 功能 | ... | ... | ... |
| 性能 | ... | ... | ... |
| 成本 | ... | ... | ... |

【各自优势】
- A: ...
- B: ...

【各自劣势】
- A: ...
- B: ...

【适用场景】
- 选 A 如果: ...
- 选 B 如果: ...

【推荐】针对具体场景的建议
```

### 注意事项
- 保持客观中立，不预设立场
- 区分客观事实和主观评价
- 考虑用户的具体场景和需求
- 承认不确定性，避免绝对化结论
